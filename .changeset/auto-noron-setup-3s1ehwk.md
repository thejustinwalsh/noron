> Branch: fix-runner-update
> PR: https://github.com/thejustinwalsh/noron/pull/3

## Changes

- Runner container is now built via `bench-runner-update --force` (with one automatic retry) instead of a direct `podman build` call; this uses the same script the weekly timer runs, ensuring parity between initial setup and subsequent updates
- New `bench-runner-update.service` and `bench-runner-update.timer` systemd units generated by `generate.ts` and installed during setup; the timer fires weekly (Sunday 04:00, randomized ±1 h) to rebuild the runner image with the latest GitHub Actions runner release
- `bench-runner-update.timer` is enabled and started as part of the tuning services block at the end of install
- New `STEPS` constants object exported from `installer.ts`: single source of truth for step names shared between the installer and the UI
- New `getInstallSteps(config, isFirstRun)` function exported from `installer.ts`: dynamically generates the ordered step list, omitting Tailscale and bootstrap-invite steps when not applicable
- Install TUI: uses `getInstallSteps()` for the initial step list, replacing the hardcoded `STEP_NAMES` array
- Critical install failures now set a `fatal` field on the result; the TUI halts progress and displays the full error with a recovery hint (`sudo bench-setup --reconfigure`)
- `runAsync()` now captures the last 20 lines of stdout/stderr and appends them to failure error messages for easier diagnosis
- Running step text colour changed from `white` to `whiteBright`; error output hidden in the step list when a fatal error is displayed
- Network step: removed erroneous `useInput` handler on the Tailscale field that was triggering `onNext` twice on Enter
- `linux-perf` and `socat` added to the `apt-get install` package list
- Lint fix: template literal used in `bench-runner-update.sh not found` error message

Replaced direct `podman build` with `bench-runner-update --force` for initial setup, added a weekly systemd timer for automatic runner image rebuilds, and improved fatal-error handling in the install TUI.
